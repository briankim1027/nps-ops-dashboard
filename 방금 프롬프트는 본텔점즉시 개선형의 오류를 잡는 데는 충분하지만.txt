방금 프롬프트는 본텔점/즉시 개선형의 오류를 잡는 데는 충분하지만, 비판매성 취약형과 구조 개선형까지 정책적으로 완전히 커버하려면 기준을 한 줄 더 명확히 줘야 해.

핵심은 이거야.

Action Card의 목표Gap은 “카드가 문제 삼는 축” 기준으로 표시해야 한다.

그래서 유형별 기준을 이렇게 정해줘야 함.

즉시 개선형
• 카드가 문제 삼는 축: 기본은 비판매성 care/risk 축, 단 비추천 원인이 판매성일 수도 있음
• 목표Gap 표시 기준: 우선 비판매성 Gap, 필요 시 종합 Gap 보조

비판매성 취약형
• 카드가 문제 삼는 축: 비판매성
• 목표Gap 표시 기준: 비판매성 Gap

구조 개선형
• 카드가 문제 삼는 축: 판매성 + 비판매성 동시 약점
• 목표Gap 표시 기준: 판매성 Gap / 비판매성 Gap 둘 다 표시

판매성 취약형, 향후 포함 시
• 카드가 문제 삼는 축: 판매성
• 목표Gap 표시 기준: 판매성 Gap

즉, 비판매성 취약형은 단순해.
무조건 비판매성 Gap이면 돼.

구조 개선형은 다름.
구조 개선형은 “양축이 같이 흔들리는 매장”이기 때문에 하나의 Gap만 보여주면 부족해.
여기는 이렇게 표시하는 게 맞아.

Gap 판매성 -12.0p / 비판매성 -37.0p · 종합 70


또는 compact하게:

판매성 Gap -12.0p · 비판매성 Gap -37.0p · 종합 70

Claude에게 줄 보강 프롬프트
아래를 추가로 붙여주면 돼.

방금 Action Card 목표Gap 수정 지시에서, 즉시 개선형뿐 아니라 비판매성 취약형/구조 개선형까지 유형별 기준을 명확히 반영해줘.

핵심 원칙:
Action Card의 `목표Gap`은 종합 NPS 고정이 아니라, “해당 카드가 문제 삼는 축” 기준으로 표시한다.

유형별 기준은 아래처럼 적용해줘.

1. 즉시 개선형
   - 기본적으로 Care Priority/Risk Map이 비판매성 축 중심이므로 비판매성 Gap을 우선 표시
   - 표시 예:
     `비판매성 Gap -37.0p · 종합 70 · 판매성 91 / 비판매성 50`
   - 단, 데이터상 비판매성 응답이 없고 판매성만 문제인 경우에는 판매성 Gap을 fallback으로 표시

2. 비판매성 취약형
   - 반드시 비판매성 Gap을 표시
   - 표시 예:
     `비판매성 Gap -37.0p · 종합 70 · 판매성 91 / 비판매성 50`

3. 구조 개선형
   - 판매성/비판매성이 함께 흔들리는 유형이므로 한 개 Gap만 표시하지 말 것
   - 판매성 Gap과 비판매성 Gap을 둘 다 표시
   - 표시 예:
     `판매성 Gap -12.0p / 비판매성 Gap -37.0p · 종합 70`
   - 가능하면 뒤에 `판매성 75 / 비판매성 50`도 같이 유지

4. 향후 판매성 취약형 카드가 포함될 경우
   - 판매성 Gap을 기준으로 표시
   - 표시 예:
     `판매성 Gap -12.0p · 종합 82 · 판매성 75 / 비판매성 91`

추가 표시 기준:
- `월누적` KPI가 종합 NPS라면 라벨을 `월누적 종합` 또는 `종합`으로 바꿔 오해를 줄여줘.
- 최근7일/오늘 KPI도 현재 종합 기준인지, 비판매성 기준인지 확인해줘.
- 만약 Action Card가 비판매성 care 중심이라면 최근7일/오늘도 비판매성 기준으로 맞추는 게 더 자연스럽다.
- 단, 기존 데이터 구조상 최근7일/오늘이 종합 기준이면 라벨에 `최근7일 종합`, `오늘
 (1/2)
종합`처럼 명확히 표시해줘.
- 본텔점 검산 기준:
  - 종합 NPS 69.6 → 종합 Gap -17.4
  - 판매성 NPS 90.9 → 판매성 Gap +3.9
  - 비판매성 NPS 50.0 → 비판매성 Gap -37.0
  - 최근7일 NPS 100
  - 오늘은 해당 최신 업무처리일 응답 없음 → null

작업 후 본텔점뿐 아니라 비판매성 취약형 1개, 구조 개선형 1개 샘플을 골라서 표시 숫자가 맞는지 검산해서 보고해줘.

구조 개선형은 반드시 양축 Gap 동시 표시를 명시해줘야 해. (2/2)
